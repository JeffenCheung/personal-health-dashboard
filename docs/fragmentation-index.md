## 设计理念

### 科学背景

碎脚指标的设计基于 2024 年发表于 **《英国皇家学会学报 B-生物科学期刊》(Proceedings of the Royal Society B: Biological Sciences)** 的同行评审研究。

#### 核心发现

| 发现 | 量化数据 | 含义 |
|------|----------|------|
| **间歇能耗增幅** | 间歇步行能量消耗可达连续步行的 **2.9 倍** | 走走停停比连续走多消耗近 3 倍能量 |
| **启动过渡成本** | 10 秒步行氧气消耗是 240 秒的 **3.4 倍** | 行走段越短，每次启动的过渡能耗占比越高 |
| **肌腱弹性回收** | 跟腱在连续运动中可回收利用弹性势能 | 停止后能量散失，无法用于下次运动 |

#### 两大能耗机制

**1. 启动过渡成本 (Start-up Transition Cost)**

人体从静止状态启动时，能量消耗显著高于稳态运动。这是因为：
- 肌肉需要额外激活能量来克服惯性
- 心肺系统需要时间达到稳态摄氧量
- 每次启动都有一个"热身"过渡期

行走段越短，这个过渡期占总运动时间的比例越高，累计额外能耗越多。

**2. 肌腱弹性损失 (Tendon Elastic Energy Loss)**

人体利用两种"节能"机制：
- **弹簧机制**：跟腱（Achilles Tendon）在脚着地时被拉长，储存弹性势能；蹬地时回缩释放能量，减少肌肉收缩所需能量
- **钟摆机制**：下肢在步行周期中类似钟摆运动，利用重力势能转换

在连续运动中，这两种机制可以高效配合，使运动更加"节能"。一旦停下来：
- 刚回收的弹性势能散失
- 肌腱回到松弛状态
- 无法将能量传递到下一次运动

因此，断断续续的行走无法利用这种"连续节能"效应，能量效率大幅降低。

> **参考来源：**
> - 果壳报道：《逛博物馆真的吸人精气？你的感觉是对的，博物馆就是会更累！》(2024)
> - 原始研究：*Proceedings of the Royal Society B: Biological Sciences*, October 2024

### 指标目标

量化一个人日常行走的"碎片化程度"，帮助用户：
1. 意识到自己的行走模式
2. 优化行走方式，提高能量效率
3. 关联疲劳感与行走模式的关系

## 核心概念

| 术语 | 定义 |
|------|------|
| **行走段**（Walk Bout） | 连续的步数记录，中间间隔不超过阈值（默认5分钟） |
| **长连续段** | 持续时间 ≥ 10分钟 的行走段 |
| **短碎片段** | 持续时间 < 3分钟 的行走段 |
| **段间隔** | 两个行走段之间的静止时间 |
| **碎脚指数** | 综合衡量行走碎片化程度的 0-100 分，分数越低越高效 |

## 数据来源

### 主要数据源
Apple Health 中的 `StepCount` 记录，每条记录包含：
- `startDate`：记录开始时间
- `endDate`：记录结束时间
- `value`：步数
- `sourceName`：数据来源（Apple Watch 优先）

### 数据粒度要求
- 理想粒度：每条记录 ≤ 1 分钟
- 可接受粒度：每条记录 ≤ 5 分钟
- 若粒度过粗（如每日汇总），无法计算碎脚指标，降级处理

## 算法公式

### 第一步：识别行走段

将当天所有步数记录按时间排序，然后合并为"行走段"：

```
输入：按时间排序的步数记录列表
输出：行走段列表

1. 初始化当前行走段 = null
2. 遍历每条步数记录：
   a. 如果当前行走段为空，创建新行走段
   b. 如果当前记录与当前行走段的间隔 ≤ 5分钟：
      - 合并到当前行走段（更新结束时间和步数）
   c. 否则：
      - 结束当前行走段，加入结果列表
      - 创建新的当前行走段
3. 遍历结束后，将最后一个行走段加入结果列表
```

### 第二步：计算子指标

对每个行走段，计算：
- **段时长** = endTime - startTime（分钟）
- **段步数** = 该段内总步数
- **段步频** = 段步数 / 段时长（步/分钟）

对全天所有行走段，计算：
- **总行走段数** = 行走段列表长度
- **长连续段数** = 时长 ≥ 10分钟 的段数
- **短碎片段数** = 时长 < 3分钟 的段数
- **平均段时长** = 总行走时间 / 总行走段数
- **总步数** = 所有段步数之和

### 第三步：计算碎脚指数

```
碎脚指数 = 碎片化占比 × 40 + 段频率 × 30 + 间隔系数 × 30

其中：
1. 碎片化占比 = 短碎片段数 / max(总行走段数, 1)
   - 范围：0 ~ 1
   - 含义：碎片段占所有行走段的比例

2. 段频率 = min(总行走段数 / 20, 1.0)
   - 范围：0 ~ 1
   - 含义：行走段越多越碎（以20段为满分基准）

3. 间隔系数 = 1 - min(平均段时长 / 15, 1.0)
   - 范围：0 ~ 1
   - 含义：平均段时长越短，碎片化越严重（以15分钟为连续基准）

最终结果范围：0 ~ 100
- 0分 = 完全连续行走，最经济高效
- 100分 = 极度碎片化，最耗能
```

### 可选：能量损耗估算

```
估算额外能耗 = 总步数 × 碎脚指数 / 100 × 0.3 kcal
```
（0.3 kcal/步 为碎片化行走相对连续行走的额外能耗估算系数）

## 评级标准

| 碎脚指数 | 评级 | 解读 | 建议 |
|----------|------|------|------|
| 0 - 20 | **A 级** | 行走模式高效，以连续步行为主 | 保持现状，非常棒的行走习惯 |
| 21 - 40 | **B 级** | 轻度碎片化，整体较连续 | 可适当减少不必要的短距离往返 |
| 41 - 60 | **C 级** | 中度碎片化，走走停停较多 | 尝试合并任务，减少频繁起身走动 |
| 61 - 80 | **D 级** | 高度碎片化，能量浪费明显 | 有意识地安排连续步行时间 |
| 81 - 100 | **E 级** | 极度碎片化，严重耗能 | 建议主动安排散步时间，减少无效走动 |

## 解读建议

### 与疲劳感关联
- 如果碎脚指数高 + 主观疲劳感强 → 可能是碎片化行走导致的
- 如果碎脚指数低但仍疲劳 → 需从睡眠、运动负荷、压力等其他方面找原因

### 不同人群的正常范围
- 办公室工作者：通常 C~D 级（频繁起身）
- 学生/教师：通常 B~C 级
- 体力劳动者：通常 A~B 级
- 跑步爱好者：跑步时段为 A 级，日常可能为 C 级

### 优化建议

1. **合并任务**：把需要走动的事情集中处理，减少来回跑
2. **定时散步**：安排固定的连续散步时间（如午饭后走15分钟）
3. **减少无效走动**：比如频繁起身拿东西，可以一次多拿几样
4. **站立办公**：如果需要频繁起身，不如站立办公减少站起坐下的能耗
5. **走路看手机**：边走路边看手机会导致步频不稳、效率降低

## 边界处理

### 数据不足
- 当天步数 < 1000 步：不计算碎脚指数，标记为"活动量不足"
- 数据粒度 > 10分钟：无法准确分段，标记为"数据粒度不足"

### 特殊情况
- 跑步数据单独处理，不计入日常碎脚指标（跑步是有目的的运动）
- 登山、爬楼等活动按行走段处理
- 游泳、骑行等非步行活动不纳入计算

### 数据异常处理
- 单条记录步数 > 5000 步且时长 < 5 分钟：标记为异常，排除
- 步频 > 200 步/分钟 或 < 40 步/分钟：标记为异常，排除
- 同一天行走段数 > 100 段：可能是数据噪声，取前100段
